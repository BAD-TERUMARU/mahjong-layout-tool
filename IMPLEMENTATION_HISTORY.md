# 麻雀牌レイアウトツール — 実装履歴・機能一覧

このファイルは、麻雀牌レイアウトツールでこれまで実装・改善した内容をまとめた記録です。

- 公開URL: https://inocchilife.github.io/mahjong-layout-tool/
- 技術: React / TypeScript / Vite / CSS
- 保存先: ブラウザ内ストレージ（画像を含む大きなデータにも対応）
- 公開方法: GitHub Pages

## 目的

麻雀の問題作成、牌姿の説明、講義資料、メモ作成に使える、ブラウザ上の自由配置ツールです。牌だけでなく、文字・図形・線・画像を一つのワークスペースに配置できます。

## 基本の牌機能

- 萬子・筒子・索子の1〜9、字牌、赤5萬・赤5筒・赤5索を牌一覧から配置
- 同一牌を複数枚配置
- クリック・ドラッグ＆ドロップで配置
- 牌のドラッグ移動、複数選択後の一括移動、削除、複製
- Delete / Backspace による削除
- ダブルクリックによる表向き・裏向きの切替
- 複数選択中の牌をまとめて裏返し
- 裏面は単一色のデザイン
- 赤ドラに不要な赤い文字・枠が出ない表示へ調整
- 牌一覧へドラッグして削除（文字、図形、画像、線も対象）
- 削除後に元に戻す操作で復元可能

## 牌の配置・整列

- 自動配置、牌を整列、ランダム13枚配牌、ランダム14枚配牌、シャッフル
- 配牌は通常牌と赤5を合わせて最大4枚になるルールで生成
- 配牌は萬子、筒子、索子、字牌の順へ理牌
- グリッド表示とグリッド吸着の切替
- 牌の間隔を `0` に統一し、2枚・3枚・6枚ごとの不要な空白を発生させないよう修正
- 牌を手動で動かしても、新規自動配置の開始位置が移動後の牌に引っ張られないよう調整
- 自動配置位置に牌がある場合は、重なりを避けて次の空き位置へ配置
- 上部に牌数を数えるための目盛り表示を追加・調整

## 文字機能

- ワークスペースの任意位置をクリックして文字を配置
- 既存文字のダブルクリック編集
- 文字の選択、解除、移動、削除、複製、コピー＆貼り付け
- 文字色、フォント、文字サイズの変更
- Word風のホームタブから文字スタイルを操作
- 文字を未選択でも新規文字の既定スタイルを変更可能
- 選択中の文字ごとに個別編集可能
- 文字の色用カラーパレットを用意

## 図形・記号機能

- 長方形（牌3枚分）、丸（牌2枚分）、三角形（牌2枚分）、バツ（牌1枚分）を配置
- 図形の選択、移動、削除、複製、コピー＆貼り付け、ロック
- 図形の色、線の太さ、サイズを変更
- 図形はドラッグ＆ドロップでの配置を基本操作に変更
- 挿入タブから配置した図形もダブルクリックで選択可能
- 記号・文字は再度選択して選択解除可能
- 配置中は実際の図形のプレビューを表示

## 線・描画ツール

- 自由線、直線、曲線、矢印を追加
- 線の色・線の太さをホームタブから変更
- 直線・曲線・矢印を記号／文字パネルと右クリックメニューに追加
- 曲線はドラッグ方向に応じて上向き・下向きに作成可能
  - 左から右へドラッグ: 上向き
  - 右から左へドラッグ: 下向き
- 線・曲線・矢印の選択表示を四角い枠ではなく、線そのものに沿う青いハイライトへ変更
- 線、曲線、矢印の保存範囲は、線幅や矢印の先端を含む最小限の大きさに調整
- 線の近くへ新しい線を描く際に、過剰な透明の四角領域が操作を妨げにくいよう改善
- 既存の保存済み線も、復元時に見た目を維持したまま選択範囲を補正

## 画像機能

- 画像ファイルをワークスペースへ配置
- 画像のドラッグ＆ドロップ、クリップボードからの貼り付け
- 画像の移動、削除、複製、コピー＆貼り付け、ロック
- 画像の外枠ドラッグによるサイズ変更
- 画像の大きなデータを含む保存へ対応

## 選択・操作性

- 空白部分のドラッグで範囲選択
- 牌、文字、図形、線、画像の複数選択
- 選択要素の一括移動・削除・複製・ロック・前後移動
- 左クリックで選択、空白クリックで選択解除
- 右クリックした要素を自動選択し、アプリ独自のコンテキストメニューを表示
- ワークスペースの空白部分でも右クリックメニューを表示
- 右クリックメニュー: 選択、複製、削除、コピー、貼り付け、文字／図形追加、表裏切替、ロック、プロパティ編集、元に戻す、やり直す
- コンテキストメニューは画面端からはみ出さないよう位置調整
- Escでメニューや配置ツールを解除
- 配置ツールは別ツールを選ぶかEscを押すまで維持し、連続配置可能

## キーボードショートカット

- Delete / Backspace: 選択要素を削除
- Ctrl または Command + C: コピー
- Ctrl または Command + V: 貼り付け
- Ctrl または Command + D: 複製
- Ctrl または Command + Z: 元に戻す
- Ctrl または Command + Y、または Ctrl / Command + Shift + Z: やり直す
- 矢印キー: 少し移動
- Shift + 矢印キー: 大きく移動
- Esc: 選択・配置モード・メニューを解除

## 履歴（Undo / Redo）

次の操作を履歴として保存し、元に戻す・やり直すに対応しています。

- 要素の配置、移動、削除、複製
- 表裏切替
- 文字編集
- 図形・画像・線のサイズ変更
- 色、フォント、線の太さなどのプロパティ変更
- 前面・背面の順序変更
- ロック状態変更

## 色・スタイル

- 文字用と図形用の色指定を分離
- 標準の色パレットを用意
- カスタム色を追加・削除できるポップアップ型カラーピッカー
- 細かい色指定に対応
- カスタムパレットは文字・図形のスタイルで共有し、追加・削除を自動同期
- 新規文字の既定色を、追加する図形にも反映
- 図形、直線、曲線、矢印にも色変更を適用

## ワークスペース

- 背景を白色へ変更し、境界・グリッドを見やすく調整
- ワークスペースは画面上部の操作エリアと牌一覧を避け、残りの表示領域を最大限使用
- サイズ入力による幅・高さ変更を一時実装した後、不要な数値入力は削除
- 拡張型ワークスペースから、実質的に制限なく使える大きなキャンバスへ変更
- パン操作に対応
- 画面上部の説明バー（WORKSPACEなどの表示）を削除して表示領域を確保
- PC・スマートフォンで操作しやすい表示へ調整

## ツールバー・画面構成

- タイトル「麻雀牌レイアウトツール」
- Wordを参考にした、ホーム・挿入などのタブ式リボン風操作エリア
- タイトルとタブ部分をコンパクト化
- ホームタブ: 文字スタイル、図形スタイル、色、線幅など
- 挿入タブ: 文字、画像、図形、線、曲線、矢印など
- 初回操作説明と、閉じられる操作ガイド
- 操作ガイドの内容を充実化

## 保存・ページ管理

- 現在の配置を自動保存し、再読み込み後に復元
- 複数の保存ページを作成
- 保存ページのタイトルを編集
- 保存ページの一覧から呼び出し、削除
- 「ブラウザ保存」という表記を「保存」へ変更
- 保存容量不足を減らすため、大きいデータに対応する保存方式へ変更
- 保存ページは、牌・文字・図形・線・画像・色・位置・回転・ロックなどを保持

## 共有・ファイル

- 保存したレイアウトを共有用ファイルとして出力
- 共有ファイルをワークスペースへドラッグ＆ドロップして読み込み
- 共有レイアウトを開くためのURL共有要素を追加
- 長すぎるURLの代わりに、共有ファイルを使う運用も可能
- JSON書き出し・読み込み・PNG保存など、不要と指定された旧操作ボタンは画面から削除

### データ設計

ワークスペースの要素は `CanvasElement` として共通管理しています。要素種別は以下です。

- `tile`: 麻雀牌
- `text`: 文字
- `symbol`: 長方形、丸、三角、バツ
- `drawing`: 自由線、直線、曲線、矢印
- `image`: 画像

各要素には原則として `id`、`x`、`y`、`rotation`、`selected`、`zIndex`、`locked` を持たせています。種類ごとの属性（牌種、文字内容、線の座標、画像データなど）は型別に追加します。

### 保存データの注意

- 現在の保存はブラウザ端末ごとのデータです。別端末・別ブラウザには自動同期されません。
- 共有には共有レイアウトファイルを利用します。ワークスペース上へドラッグ＆ドロップすると読み込めます。
- 画像はデータURLとして保存されるため、保存サイズが大きくなります。保存層は大きいデータ向けに対応済みですが、極端に大きい画像を大量に置く場合は縮小してから使うと安全です。
- 保存形式を変更する場合は、`App.tsx` の `parseElement` と保存データの移行処理を必ず確認します。

### 牌画像の差し替え

- 牌の定義は `src/data/tiles.ts` に集約されています。
- 画像を交換する場合は、その定義ファイルのアセット参照を変更します。
- 牌の画像そのものをコンポーネントへ直接書かない設計です。
- 牌サイズの基準値は `src/utils/layout.ts` の `TILE_WIDTH` と `TILE_HEIGHT` です。画像、目盛り、図形サイズ、整列に影響するため、変更時は一緒に確認します。

## 引継ぎ用: 開発・確認手順

### 必要環境

- Node.js
- pnpm
- Git
- GitHub CLI（公開更新を行う場合）

### ローカル起動

```bash
pnpm install
pnpm dev
```

### 確認コマンド

```bash
pnpm run lint
pnpm run build
pnpm preview
```

変更後は、少なくとも `pnpm run lint` と `pnpm run build` を通します。操作変更を行った場合は、ブラウザで対象の操作（配置、選択、移動、保存、再読み込み）も確認します。

### GitHub Pagesへの公開

このリポジトリは `main` ブランチへのpushを契機に、GitHub ActionsでGitHub Pagesへデプロイします。

```bash
git add <変更ファイル>
git commit -m "変更内容"
git push origin main
```

デプロイ後は、GitHub Actionsの **Deploy to GitHub Pages** が成功したことを確認します。Viteの公開先設定は `vite.config.ts` の `base: '/mahjong-layout-tool/'` です。リポジトリ名を変更する場合は、この値も変更します。

## 引継ぎ時に確認すること

1. `pnpm install` 後に lint と build が通ること
2. 牌の配置・移動・削除・表裏切替が動くこと
3. 文字、図形、線、画像の追加・選択・プロパティ変更が動くこと
4. Undo / Redo とコピー＆貼り付けが動くこと
5. 保存、再読み込み、共有ファイルのドラッグ＆ドロップ読み込みが動くこと
6. GitHub Pagesの公開URLで画面が表示されること

## 運用上の補足

- 既存の要望で「作業完了時は公開更新まで行う」運用になっています。今後も機能変更を完了した際は、ビルド確認・GitHubへのpush・GitHub Pagesの成功確認まで実施してください。
- 既存機能を変更する際は、牌の保存・読み込み・複数選択・ドラッグ削除・右クリックメニューを壊さないよう、関連操作をまとめて確認してください。
